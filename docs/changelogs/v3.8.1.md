# v3.8.3 (Hardened Stability)

**Release Date:** 2026-04-17

This patch addresses critical synchronization bottlenecks and audit logging integrity issues observed in high-load legacy environments.

---

## 🛡️ Connection Hardening
- **Handshake Expansion**: Increased `connectTimeoutMs` and `defaultQueryTimeoutMs` to **90 seconds** to accommodate slower hardware processing.
- **Sync Throttling**: Explicitly disabled `syncFullHistory` and `shouldSyncHistoryMessage` to prevent "Timed Out" errors during initial connection handshakes.
- **No-op Message Handler**: Implemented a lightweight `getMessage` callback to satisfy Baileys' internal sync requirements without memory overhead.

## 📊 Audit Trail Integrity
- **Log Noise Filtering**: Silenced decryption errors specifically for `status@broadcast` to reduce log bloat and focus on transactional failures.
- **Direct Dispatch Reliability**: Fixed a race condition where instant dispatches would fail without updating the audit table. All direct dispatches now default to a 'Failed' state placeholder which is overwritten to 'Success' only upon verified socket delivery.
- **Webhook Silence**: Removed the default webhook placeholder to stop 404 error loops in the system logs.
