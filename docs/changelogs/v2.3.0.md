# Changelog v2.3.0 - Resilience Engine

**Release Date:** 2026-04-10
**Summary:** Major resilience update introducing automated recovery for temporary network failures.

### 🛡️ Core Resilience
- **Exponential Backoff Retry**: Added a sophisticated retry loop to message dispatch. The bridge will now automatically retry failed messages up to 3 times (by default) with increasing delays (5s, 15s, 60s).
- **Retry Tracking**: Integrated `retry_count` into the SQLite audit logs, allowing for full visibility into message delivery struggles.
- **Configurable Thresholds**: Introduced `MAX_RETRIES` environment variable support for fine-tuning performance.

### 🏗️ Architecture
- **Version Bump**: Increment to v2.3.0, marking the completion of the "Priority 1" roadmap for Phase 2.
