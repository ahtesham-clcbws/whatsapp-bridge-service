# v3.8.5 (Stabilization & Diagnostics)

**Release Date:** 2026-04-17

The v3.8.5 release is a critical maintenance patch designed to harden the bridge for long-term uptime on legacy hardware. This version prioritizes data integrity, memory optimization, and transparency.

---

## 🛠️ Stabilization & Core Fixes
### 📡 Presence Warm-up (iPhone E2E Fix)
Implemented `sendPresenceUpdate('composing')` before every message dispatch. This forces the recipient's device to perform an encryption handshake, resolving the "Waiting for this message" error commonly seen on idle iPhone sessions.

### 🗄️ SQLite Connection Singleton
Refactored the database layer to utilize a cached connection promise. This prevents "Too many open files" errors and race conditions during high-volume starts (e.g., when 500+ offline messages arrive at once).

### 🧠 BLOB Media Optimization
Binary data (Images/PDFs) is now stored directly as **SQLite BLOBs** rather than stringified JSON buffers. This drastically reduces RAM usage on 2-core machines like the 2014 MacBook Air.

---

## 🚨 Failure Diagnostics Dashboard
A new dedicated control plane for managing delivery obstacles.

- **Technical Audit Log**: A filtered view of the last 100 failed messages with the specific socket error code.
- **Manual Retry Hub**: One-click re-enqueueing of failed messages, allowing administrators to recover from temporary device downtime.
- **Maintenance Heartbeat**: The 30-minute heartbeat now triggers recursive data purging to keep the database lean.

---

## 🍎 Legacy Environment Patches
- **macOS Big Sur Compatibility**: Patched building warnings and installation instructions for legacy Apple hardware.
- **Diagnostic Tool**: Introduced `.agent/scripts/check-env.sh` to verify system readiness.
